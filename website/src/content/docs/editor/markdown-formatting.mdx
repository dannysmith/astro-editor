The editor is a **markdown source editor** not a preview, so you always see the actual characters in your file. Syntax highlighting is mostly just dimming markers like `**` so they don't distract from the words you're writing, and subtly styling some elements so documents are easier to scan. Common edits have the keyboard shortcuts you'd expect (see [Keyboard Shortcuts](/reference/keyboard-shortcuts/)).

## Headings

Headings all rendered the same, regardless of level. When there's room, hash-style heading markers "hang" in the left margin so the text stays aligned with the body copy. Heading colour is one of the few [configurable preferences](/preferences/) because it can hugely improve scanability when working with long documents.

<AEDemo code={`# Heading 1
## Heading 2
### Heading 3`} />

You can transform the current line into a heading with the same shortcuts as in [Notion](https://notion.so).

| Shortcut | Action |
| --- | --- |
| <Kbd mac="Opt+Cmd+1" windows="Alt+Ctrl+1" />–<Kbd mac="Opt+Cmd+4" windows="Alt+Ctrl+4" /> | Set the line to a Heading 1–4 |
| <Kbd mac="Opt+Cmd+0" windows="Alt+Ctrl+0" /> | Remove the heading (back to a paragraph) |

## Bold & Italic

Wrap text in `**` for bold and `*` for italic.

<AEDemo code="this is **bold** and this is *italic*" />.

| Shortcut | Action |
| --- | --- |
| <Kbd mac="Cmd+B" windows="Ctrl+B" /> | Wrap the selection in `**` (bold) |
| <Kbd mac="Cmd+I" windows="Ctrl+I" /> | Wrap the selection in `*` (italic) |

Both shortcuts toggle — run them again on the same selection to remove the formatting.

## Lists

Unordered lists start with `-`, `*`, or `+`; ordered lists start with a number followed by a dot. Just start typing — there's no shortcut needed.

<AEDemo code={`- First item
- Second item

1. First step
2. Second step`} />

## Links

Write links with standard markdown, or use <Kbd mac="Cmd+K" windows="Ctrl+K" /> to wrap the selected text in a link and drop your cursor into the URL. Hold <Kbd mac="Opt" windows="Alt" /> and click any link to open it in your browser.

<AEDemo code="[Astro Editor](https://astroeditor.danny.is)" />

To link to another item in your content collections, use the Content Linker (<Kbd mac="Cmd+Shift+K" windows="Ctrl+Shift+K" />) — see [Links](/editor/links/) for more.

## Inline Code & Code Blocks

Wrap inline code in single backticks; it's shown in Astro Editor's monospace font with a subtle background. Fenced code blocks (three backticks) get a tinted background across the whole block.

<AEDemo code={`Use the \`glob()\` loader in your config.

\`\`\`ts
const articles = defineCollection({ /* ... */ })
\`\`\``} />

Code **inside** fenced blocks is deliberately not syntax-highlighted by language — Astro Editor is a writing tool, not a coding one. See the [Philosophy](/getting-started/philosophy/) page for why.

## Blockquotes

Start a line with `>`. The whole quoted line is shown in grey italics.

<AEDemo code="> A quote stands out from the surrounding text." />

## Other Syntax

The editor also styles images (`![alt](src)` — hold <Kbd mac="Opt" windows="Alt" /> and hover to [preview them](/editor/images-and-files/)), horizontal rules (`---`), and inline HTML.
